1. Create the venv

In your project root (my-weather-app/):

python -m venv .venv


This creates a hidden .venv/ folder that contains an isolated Python + pip.

2. Activate the venv

Activation differs by OS/shell:

macOS / Linux (bash/zsh):

source .venv/bin/activate


Windows PowerShell:

.venv\Scripts\Activate.ps1


Windows CMD:

.venv\Scripts\activate.bat


After activating, you should see (.venv) at the start of your shell prompt.

3. Install dependencies inside venv

Now that your environment is isolated, install your project requirements:

pip install --upgrade pip
pip install -r requirements.txt


You can confirm by running:

pip list


You should see requests and python-dotenv installed inside .venv.

4. Run your script inside venv

Make sure your .env file exists at the project root:

WEATHER_API_KEY=your_real_api_key_here


Run the weather app:

python src/weather.py London

5. Running tests inside venv

Install pytest inside the venv if you haven’t already:

pip install pytest


Then:

pytest -q

6. Deactivate venv

When you’re done working:

deactivate


This brings you back to your system Python.
